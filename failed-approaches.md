# Failed Approaches — 共通失敗原則集

本ファイルは過去のイテレーションで試した改善案の失敗から抽出された **汎用的な原則** のみを記載する。
具体的なベンチマークケース ID、特定 iter 番号、特定の失敗ケースに関する記述は含まれない。

（meta-agent-2/auto-improve ブランチ開始に伴いリセット。旧原則は failed-approaches-v1.md に保存。）

## 原則

1. 再収束を比較規則として前景化しすぎない
- 内部の経路差を「次の共有観測点で再収束するか」で扱う規則を強化すると、比較が下流の一致に過度に引っ張られ、途中の差分シグナルを弱めすぎることがある。
- 特に EQUIVALENT 側で「最初の差分」と「それを吸収する後段処理」の対提示を既定化すると、差分の検出よりも吸収の説明を先に組み立てる読み方を誘発しやすい。
- 特に Guardrail へ追加すると、局所的な差を provisional 扱いする方向に過剰適応し、判別力よりも保守性が先行しやすい。
- 失敗は「比較粒度の精密化」そのものではなく、「再収束を優先する規範を新たな既定動作にしたこと」で起きる。

2. 未確定な relevance や脆い仮定を、常に保留側へ倒す既定動作にしすぎない
- relevance 未解決時に「除外するまで暫定採用」や「まず UNVERIFIED へ送る」といった既定分岐を追加すると、未確定性そのものが強い判断信号になり、必要以上に保守的な比較へ寄りやすい。
- 結論直前に「未検証の最弱リンクが verdict を左右しうるなら確定しない」のような不安定性チェックを Guardrail 化しても、仮定の弱さ評価それ自体が保留トリガーになり、局所確認より先に非確定化へ最適化されやすい。
- 同様に、比較ごとの証拠の弱い側を必ず特定させ、その側の未検証性を次の必須行動に結びつけると、比較そのものより不確実側の救済が既定化し、局所的な証拠非対称を過大評価しやすい。
- verdict を左右する claim ごとに「未検証依存なら結論前に非確定化する」といった判定を必須化しても、局所探索の促進より先に非確定化そのものが安全策として学習され、比較判断の出力が痩せやすい。
- direct な根拠が薄い場面で、間接的な呼び出し元・import 元・再公開元まで探索し尽くすまで relevance を閉じない既定動作を足すと、弱い到達可能性を広く暫定採用する方向に寄りやすい。
- その結果、比較の前提となる関連範囲が不必要に膨らみ、局所的な有力証拠より「まだ拾えるかもしれない間接経路」の救済が優先されやすい。
- 結論直前に重複した self-check や禁止事項を足し、未解決項目が 1 つでもあれば再探索へ戻す構造にすると、新しい証拠生成より checklist 充足が優先され、局所的で非決定的な不確実性まで広く保留側へ倒れやすい。
- これは 1 と同様に、差分を丁寧に扱ったこと自体ではなく、未解決状態での fallback を Guardrail 化したことで過剰適応を招く失敗として理解できる。
- 比較欄で未検証の結果を verdict 証拠として扱わないよう明示する変更も、局所的な語彙修正のつもりでも、実際には「未検証なら比較ラベルを避ける」という保留側の既定分岐を強めうる。
- 特に既存テンプレートの二択ラベルを、確定結果が揃った場合だけ使える条件付きラベルへ狭めると、未知の扱いを改善するより先に、比較判断そのものを非確定化する圧力が増えやすい。
- 曖昧さへの対処は局所的な補助手順に留め、広い既定動作としては追加しない方がよい。

3. 差分の昇格条件を新しい抽象ラベルや必須の言い換え形式で強くゲートしすぎない
- 差分をいったん「外部に見えるものだけ」に分類してから比較証拠へ昇格させる規則を前面化すると、途中の差分シグナルを早い段階で弱め、必要な追跡まで始まりにくくなることがある。
- 差分ごとに特定の premise/assertion へ結びつけた CLAIM 形式への言い換えを必須化してから判定させる変更も、比較そのものより再記述の整合を先に満たす読み方を誘発しやすい。
- とくに Guardrail や必須チェックに載せると、観測可能性の判定や言い換えの妥当化そのものが先に目的化し、比較よりも予備分類・再記述の整合に最適化されやすい。
- 構造差の役割を「探索の手掛かり」に限定し、明示的な traced divergence が出るまで結論への利用を禁じる形も、差分の昇格条件を強くしすぎる変種になりうる。
- 高情報量の差分まで一段低い暫定シグナルへ固定すると、比較は探索順制御には慎重でも verdict の判別力を失いやすい。
- また、詳細 tracing の開始点を特定の assertion/check など単一のアンカーに固定し、その地点からの逆向き説明を既定化する変更も、比較対象そのものより説明の起点整合を先に最適化しやすい。
- side ごとの結果予測より先に観測対象を必ず固定させる順序変更も、この変種になりうる。観測点の共有化は比較を揃える一方で、最初に選んだ局所 check への過剰適応を招き、上流差分や別の判別経路を拾う柔軟性を落としやすい。
- この種の変更は「判定に効く最短経路」を狙っていても、上流の高情報量な差分や別経路の識別信号を拾う前に探索を狭め、局所的な証拠へ過剰適応しうる。
- また、見つけた差分を新しい抽象ラベルへ二分し、そのラベルで比較範囲の拡大・維持を決める形式も、差分そのものの比較より分類整合を先に満たす読み方を誘発しやすい。
- その結果、比較粒度を上げる意図でも、実際には中間ラベルの妥当化が主目的化し、直接の証拠照合が弱まって判別力を落としうる。
- 結果差を示す欄を、最初の分岐点から終端観測までの因果 chain の提示へ置換する形も、証拠品質を上げる意図に反して trace 再構成の整合を新しい昇格条件にしやすい。
- 特に反例の成立条件へ組み込むと、既に十分な結果差の比較より「最初の分岐点」を名指す作業が先に目的化し、局所 trace へ過剰適応して判別力を落としうる。
- これは 1 の「再収束を既定化しすぎる」失敗に近く、差分を直接比較する代わりに別の抽象フィルタや中間表現を追加したことで判別力を落とす一般形として整理できる。

4. 終盤の証拠十分性チェックを、単なる confidence 調整へ吸収しすぎない
- 結論直前の独立した self-check や証拠十分性確認を削り、残る未解決事項を広く CONFIDENCE へ送る設計にすると、比較の停止条件が早まり、trace の不足や検証漏れまで「非決定的な不確実性」として飲み込みやすい。
- とくに file inspection・traceability・主張範囲の抑制をまとめて担う終盤チェックを外すと、追加探索のノイズを減らす代わりに premature closure が増え、Compare の判別力を落としうる。
- これは 2 の「保留側への過剰適応」の逆側の失敗であり、未解決事項を減らしたこと自体ではなく、verdict を支える最低限の検証フロアまで confidence 記述へ移してしまったことが問題である。
- 広い「結論が証拠を超えない」確認を、最弱リンクの名指しと confidence / UNVERIFIED への振り分けに置換する形も危うい。過剰な保留を避ける意図でも、証拠十分性の総合確認が弱まり、局所的な弱点の扱いだけが目的化しやすい。

5. 最初に見えた差分から単一の追跡経路を即座に既定化しすぎない
- 意味差分を見つけた直後に、同一の経路や単一の共有テストでの paired trace を最優先の必須手順として固定すると、比較がその最初の局所シグナルへ早くアンカーされ、広い構造差や別経路の判別情報を拾う前に探索が狭まりやすい。
- これは「局所 trace を増やした」こと自体ではなく、「次に何を見るか」を一つの近傍観測へ強く固定したことで、比較の探索順が硬直化する失敗である。
- 既存の 3 と同様、差分を見たらただちに単一アンカーへ収束させる規則は、一般には探索の柔軟性と判別力を落としうる。

6. 近接する欄の統合で、探索理由と反証可能な情報利得を同じ短い要求に潰しすぎない
- 認知負荷削減を狙って、次に読む理由と、その行動がどの不確実性・仮説を解くかを一つの欄へ圧縮すると、表面上は簡潔でも反証可能性の明示が弱まりやすい。
- 特に optional な補助欄を削除して required 欄へ吸収する形では、モデルが「理由らしい一文」を満たすだけで、どの仮説が確認・反証されるかを保持しないことがある。
- 逆に、情報利得を verdict 反転可能性だけで名指させる形へ置換しても、探索が現在の結論候補に早く従属し、まだ verdict claim として定式化できない有力証拠を拾いにくくなる。
- その際に証拠セクションの完遂要求まで弱めると、必要な trace と単なる追加探索の区別が曖昧になり、confidence-only として早く閉じる方向へ寄りやすい。
- optional な情報利得欄を、読む前に最小の判別対象を名指す必須欄へ置換する形も危うい。反証可能性を強める意図でも、探索理由が「どの artifact を選ぶか」の正当化へ寄り、仮説間の不確実性を柔軟に保持しにくくなる。
- その結果、広すぎる探索を避けるつもりが、事前に選んだ判別対象への過剰適応や、見つからない場合の形式的な一段拡張に流れ、別経路の高情報量な証拠を拾いにくくなりうる。
- これは 4 の premature closure とも 5 の探索順固定とも近く、手順量を減らしたこと自体ではなく、探索を判定へつなぐ中間表現まで圧縮・単一化したことで判別力を落とす失敗である。
