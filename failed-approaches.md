# Failed Approaches（試行済み・失敗した改善方向）

本ファイルは過去のイテレーションで試行し、スコアを悪化させた改善方向を記録する。
新たな改善案を立てる際に必ず参照し、同じ方向の変更を繰り返さないこと。

## ブラックリスト

### BL-1: テスト削除を ABSENT として定義追加
- 試行: iter-6, 10, 15
- 内容: 削除されたテストは FAIL ではなく ABSENT として扱う定義（D3等）を追加
- 結果: NOT_EQ の正答率が低下（90% → 70%）
- 原因: ABSENT 定義が NOT_EQ 判定のハードルを上げすぎ、本来 NOT_EQ であるケースを YES に倒した
- Fail Core: この変更の本質は「比較対象から特定のテストを除外するルールの追加」である。テストを比較対象から除外すること自体が NOT_EQ の証拠を減らす方向に作用する。つまり「何を比較しないか」を定義する変更は、比較の網羅性を下げ、差異の検出力を弱める。EQUIV の誤判定を防ごうとして「これは比較しなくてよい」というルールを導入すると、本来比較すべきケースまで除外してしまう。

### BL-2: NOT_EQ 判定の証拠閾値・厳格化
- 試行: iter-7, 8, 13, 16, 17, 19, 20
- 内容: counterexample にアサーションまでのトレースを要求、仮想環境での反例を禁止、call path の明示を要求、等
- 結果: スコア 65-75% で安定せず
- 原因: NOT_EQ と判定すべきケースで証拠不十分として YES に倒れる。真の NOT_EQ の検出力を弱めた
- Fail Core: この変更の本質は「NOT_EQ と結論するための立証責任を引き上げる」ことである。立証責任が高いほど、エージェントは証拠が十分でないと判断して YES（EQUIV）に倒れやすくなる。NOT_EQ の閾値を上げることと EQUIV の閾値を下げることは表裏一体であり、一方を厳しくすれば他方に判定が流れる。モデルの推論能力が一定である以上、閾値の操作はトレードオフを移動させるだけで、全体の精度を上げることにはならない。

### BL-3: UNKNOWN 回答の禁止
- 試行: iter-8, 9, 11, 21
- 内容: UNKNOWN は許さず LOW confidence でも YES/NO を出すよう強制
- 結果: 改善効果なし
- 原因: 強制的に回答させても判定精度は上がらない。むしろ根拠の薄い判定が増える
- Fail Core: この変更の本質は「判断を保留する選択肢を奪い、不確実な状況でも結論を強制する」ことである。UNKNOWN が出るのはエージェントが十分な証拠を収集できなかった結果であり、そこで結論を強制しても判断材料が増えるわけではない。問題は「結論を出せないこと」ではなく「十分な証拠を集められないこと」にある。結論の強制は症状への対処であり、原因（証拠収集プロセスの質）への対処ではない。

### BL-4: 早期打ち切り（CONVERGENCE GATE）
- 試行: iter-12, 14
- 内容: 確信が得られた時点で探索を打ち切る STOP / CONVERGENCE GATE の追加
- 結果: 改善効果なし
- 原因: 探索を短縮すると見落としが増え、精度に寄与しない
- Fail Core: この変更の本質は「探索量を減らして効率を上げようとする」ことである。しかしこのタスクの誤判定は探索不足（見落とし）から生じるケースが多く、探索を早く打ち切ることは問題を悪化させる。「十分に見た上で間違える」のではなく「十分に見ないから間違える」場面では、探索の効率化ではなく探索の質の向上が必要。確信度で打ち切ると、偽の確信（誤った思い込み）を確定させてしまう。

## 共通の失敗パターン

上記4つの Fail Core から導出される共通原則:

1. **判定の非対称操作は必ず失敗する**: EQUIV か NOT_EQ の一方に有利な変更は、他方の正答率を犠牲にする。判定閾値の移動はゼロサムゲームであり、全体精度は改善しない。
2. **出力側の制約は効果がない**: 「こう答えろ」「こう答えるな」という出力への制約（UNKNOWN 禁止、証拠閾値、比較対象除外）は、推論の質自体を改善しない。改善すべきは入力側（何を見るか）と処理側（どう考えるか）である。
3. **探索量の削減は常に有害**: このタスクでは情報不足が誤判定の主因であり、探索を減らす変更は全て悪化する。
4. **同じ方向の変更は表現を変えても同じ結果になる**: BL-1 の「ABSENT 定義」も BL-2 の「証拠閾値」も、本質は「NOT_EQ を出しにくくする」という同一の効果を持つ。表現やメカニズムが異なっても、効果の方向が同じなら結果も同じ。
