# Iteration workflow on `main`

This repository is now consolidated to a single branch (`main`). New SKILL ideas
are tested directly on `main` with manual curation, then either committed or
discarded. Long-running auto-improve branches are not used anymore — they
caused the consolidation chaos that the `archive/*` tags now preserve.

## What stays out of git

The following are generated by benchmark / auto-improve runs and are
`.gitignore`d so they never pollute `main`:

- `benchmark/swebench/runs/iter-2/`, `iter-3/`, ... (only `iter-1` is committed
  as a reference baseline)
- `benchmark/swebench/runs/archive.jsonl`
- `benchmark/swebench/runs/auto-improve-*.log`
- All `compare-pro-*/`, `audit-security-*/`, `localize-*/`, `variance-*/` run
  directories

## Manual iteration (recommended for new ideas)

1. Edit `SKILL.md` directly on `main` with the new idea.
2. Run the benchmark you want to validate against, e.g.:
   ```bash
   bash benchmark/swebench/run_benchmark_compare_pro.sh
   bash benchmark/swebench/run_benchmark_audit.sh
   ```
3. Compare the result against the baseline numbers in `README.md` (Compare Pro
   64.2%, Audit-Improve 88.4%).
4. If the change is an improvement: `git add SKILL.md && git commit && git push`.
5. If not: `git checkout SKILL.md` to discard, and try a different idea.

The principle: every `main` commit reflects a manually verified improvement,
not raw automation output.

## Automated iteration (`auto-improve.sh`)

`auto-improve.sh` still runs the full propose / discuss / implement / audit /
benchmark loop, but it is now neutered for git pushing. By default it:

- Creates local commits for each iteration (so you can review and reset).
- **Never pushes to `origin`** unless `AUTO_IMPROVE_PUSH=1` is exported.

Typical session:

```bash
# Run 10 iterations locally — nothing reaches origin.
./auto-improve.sh -n 10

# Inspect the iter-* commits.
git log --oneline -20

# Pick the SKILL.md you want and reset main back to the chosen commit.
git reset --hard <commit-of-best-iter>

# Push only the curated state.
git push origin main
```

If you really want auto-push (e.g. on a dedicated experiment machine), set
`AUTO_IMPROVE_PUSH=1` explicitly. Do not enable it on a developer workstation
that pushes to `origin/main`.

## Rolling back a bad iteration

Local-only commits from `auto-improve.sh` can be discarded with:

```bash
git reset --hard origin/main      # drop all local iter-* commits
# or, more surgically:
git reset --hard <last-good-sha>
```

Once a commit is pushed to `origin/main` it should be reverted, not reset:

```bash
git revert <bad-sha>
git push origin main
```
